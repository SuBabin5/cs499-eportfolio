Algorithms and Data Structures Enhancement

Artifact Overview

The selected artifact for this category was originally developed in CS-300 (Data Structures and Algorithms). The project implemented a Binary Search Tree (BST) to store and retrieve structured data efficiently.

The original implementation successfully inserted, searched, and displayed data using BST logic. However, additional analysis and refinement were applied to strengthen algorithmic reasoning and evaluation.

Enhancements Implemented

  • Expanded documentation explaining algorithm logic
  • Analyzed time complexity for insertion and search operations
  • Discussed trade-offs between different data structure choices
  • Improved readability and structural clarity of code

Algorithmic Evaluation

The Binary Search Tree provides average-case time complexity of O(log n) for insertion and search operations when balanced. However, in worst-case scenarios where the tree becomes unbalanced, performance degrades to O(n).

Alternative data structures such as vectors or unsorted lists would require linear search with O(n) time complexity for lookups. Therefore, the BST was selected to improve efficiency as dataset size increases.

This evaluation demonstrates understanding of computational trade-offs, scalability considerations, and structured algorithm design.

Design Trade-Offs

While the BST improves average-case efficiency, it introduces potential imbalance risks without self-balancing mechanisms such as AVL or Red-Black Trees. The chosen design reflects a balance between conceptual clarity and performance appropriate for the scope of the original assignment.
