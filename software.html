Software Design and Engineering Enhancement

Artifact Overview

The selected artifact for this category was originally developed in CS-320 (Software Testing, Automation, and Quality Assurance). It consisted of object-oriented classes representing application entities and associated services responsible for managing those objects.

The original implementation met functional requirements but presented opportunities for structural improvement, enhanced validation logic, and clearer separation of responsibilities.

Enhancements Implemented

  • Strengthened input validation to prevent invalid object states
  • Improved encapsulation by limiting direct access to class fields
  • Refactored logic to improve clarity and maintainability
  • Enhanced documentation for readability and professional presentation
  • Applied iterative testing to verify functionality after modifications

Design Improvements

The refactoring process focused on reinforcing object-oriented principles such as encapsulation, separation of concerns, and controlled state mutation. By centralizing validation logic within constructors and update methods, the design prevents invalid data from entering the system.

These improvements reflect industry-standard development practices where maintainability, clarity, and defensive programming are prioritized.

Security and Validation Considerations

A security mindset was incorporated by enforcing explicit validation rules. Input values are checked before being stored, preventing malformed or inconsistent data from compromising system integrity.

This approach demonstrates awareness of adversarial or unintended misuse and aligns with best practices in secure software development.

Alignment to Course Outcomes

  • Design and evaluate computing solutions using structured software engineering practices
  • Apply iterative testing and refactoring techniques
  • Communicate design decisions clearly and professionally
  • Develop a security-aware approach through defensive programming
